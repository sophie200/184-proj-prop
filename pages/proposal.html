Summary

Our goal is to make beautiful fractals using raymarching and distance estimators. We will use shading techniques like Blinn-Phong and hopefully re-create images like the "anomaly" and "hexcore" imagery from Arcane, which we believe was implemented using some form of fractals. We will create a front end that allows users to play around with parameters and generate these fractals themselves.

Problem Description

Fractals are geometric patterns that repeat themselves and form beautiful, self-similar structures at various scales. While traditional raytracing techniques require a well-defined mesh, trying to render such a complex scene with fractals would be highly inefficient due to the sheer number of primitives involved. Therefore, with raymarching, we can use distance estimators to specify the type of fractal we want to generate and efficiently trace the path of light through the scene. Instead of relying on a vast mesh of polygons, raymarching makes it possible to render intricate fractal shapes with significantly reduced computational costs, opening up possibilities for stunning, high-quality visualizations of these complex structures. Paired with shading techniques like Blinn-Phong, we could create beautiful scenes akin to Across the Spider-Verse.

Goals and Deliverables

What We Plan to Deliver

We plan to deliver a collection of three different types of fractals: iterated function systems (e.g. Apollian gasket), escape-time fractals (e.g. Mandelbot fractals), and strange attractors (e.g. Clifford attractors). These are the types of images we hope to create:

We plan to create an interactive demo with these fractals, where, from a dropdown menu of options, a user can play around and create different types of fractals themselves! (More details described in What We Hope to Deliver)

To measure the quality of our generated fractals, we plan to use several metrics.

  • Fractal dimension: This measures the complexity of the fractal. We can estimate the fractal dimension using the box-counting method, in which we divide the image into a grid and find how many boxes are needed to cover the fractal.
  • Self Similarity: We can use SSIM (Structural Similarity Index Measure) to compare the similarity of fractals at each recursive level. This will give a measure of whether our algorithm is generating valid fractals.
  • Lyapunov Exponent: Our final metric is used to quantify the chaotic behavior of our generated fractal. Specifically, it measures the sensitivity of our fractals to changes in the initial condition. This is especially useful in assessing the quality of our generated Clifford attractors and escape-time fractals.

What We Hope to Deliver

Aside from creating the images, we hope to deliver an interactive page where the user can specify the type of fractal and type of shading and see the fractals being created in realtime. We will have various parameters, like the number of iterations, color scheme, etc., available for users to play around with.

If we get ahead of schedule, we hope to expand our fractals into 3D shapes. Most fractals are 2D, so converting to 3D requires a translation matrix or converting the fractal to 4D and then taking a 3D "slice" of the fractal. Our reach goal would be trying to recreate the "anomaly" in Arcane (below), which we believe to be a 3D version of the apollonian gasket fractal (which we plan to implement in 2D in Week 2).

Schedule

Week 1

We will spend the first week working on rendering our first fractal. This requires us to implement a ray marcher and our shading system and color scheme, which we will use for the remaining fractals that we generate. We have found existing implementations of raymarchers in C++ that we could use as a starting point. The first type of fractal we plan to create is an iterated function fractal, which is generated with geometric rewrite rules. We've chosen the apollonian gasket fractal, which mimics a pattern seen in soap bubbles by generating new circles with every iteration.

Week 2

The second type of fractal we plan to create is an escape-time fractal, which are systems that recurisvely iterate over a given input and outputs results that are within some given bound. A common escape-time fractal that we would probably first tackle is the Mandelbrot fractal, which is a fractal that has iteratively smaller and smaller versions of itself. Mathematically speaking, an escape-time fractal applies a formula to a complex number (the number starting from some pixel) and outputs a color depending on how quick that number "escapes" tto infinity.

Week 3

Our final fractal is the Clifford Attractor (a type of strange attractor), which are systems of equations that evolve over time (largely due to the inclusion and behavior of sine and cosine within the system). Additionally, if any of our Week 1 and 2 tasks take more time than expected, Week 3 could also be the week where we catch up on work.

Week 4

We will then create the frontend for our website, which will allow users to interact with the program by modifying parameters to create a unique fractal. We suspect we may need to implement the front end as a separate React app (or some other framework) and add end points to our C++ repository, however we plan to flesh out this idea further once we establish our C++ codebase.
